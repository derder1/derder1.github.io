类变量和类方法

类变量内存布局

不管static变量在哪里，共识（1）static变量是同一个类所有对象共享（2）static变量，在类加载的时候就生成了。

类变量

类变量也叫静态变量/静态属性，是该类的所有对象共享的变量，任何一个该类的对象去访问它时，取到的都是相同的值，同样任何一个该类的对象去修改它时，修改的也是同一个变量

定义语法
访问修饰符 static 数据类型 变量名;（推荐）
static 访问修饰符 数据类型 变量名;

如何访问类变量
类名.类变量名
或者 对象名.类变量名
推荐使用 类名.类变量名

细节

  1. 当某个类的所有对象都共享一个变量时，考虑使用类变量（静态变量）比如定义学生类，统计学生交多少钱。
    类变量与实例变量区别
  2. 类变量是该类的所有对象共享的，而实例变量是每个对象独享的。
  3. 类变量遵循访问修饰符限制的访问权限
  4. 类变量类加载时就创建了，即使没有创建对象，只要类加载了，就可以使用类变量了
  5. 类变量生命周期随着类的加载开始，随着类的消亡而销毁，与对象无关
  6. 实例变量（普通属性）不能通过类名.变量名方式访问

main语法说明

  1. main()方法是虚拟机调用
  2. java虚拟机需要调用类的main()方法，所以该方法权限必须是public
  3. java虚拟机在执行main()时不必创建对象，所以必须是public
  4. 该方法一个接收String类型的数组参数，该数组中保存执行java命令时传递给所运行的类的参数
  5. 比如：java 执行的程序 参数1 参数2 参数3
  1. 在main()方法中可以直接调用本类中的静态成员，不可以直接调用非静态成员

代码块

代码块又称为初始化块，属于类中的成员，属于类的一部分
和方法不同，它无方法名，无返回值，无参数，只有方法体，不用通过对象或类显式调用，而是加载类时隐式调用。

语法
【修饰符】{
代码
};

  1. 修饰符可选，但写只能写 static

  2. 代码块分两类

    • static修饰：静态代码块
    • 无static修饰：非静态代码块/普通代码块

  3. 逻辑语句可以为任何逻辑语句（输入、输出、方法调用、循环、判断等）

  4. “;”可写可不写

  • 相当另一种形式的构造器（堆构造器的补充机制），可以做初始化操作
  • 如果多个构造器有重复语句，这时可以将重复语句抽离写在代码块里

当我们不管调用哪个构造器，创建对象，都会先调用代码块的内容
代码块调用的顺序优先于构造器..

细节

  • static代码块，对类初始化，随类加载执行，并且只执行一次；

  • 无static 普通代码块，每创建一个对象执行一次

类什么时候加载(重要)

  1. 创建对象实例
  2. 创建子类对象实例，父类被加载（父类先加载，子类后加载）
  3. 直接使用类的静态成员（静态属性、静态方法）

普通的代码块，在创建对象时调用的，会被隐式的调用，被创建一次，就调用一次。如果只是使用类的静态成员，不会调用普通代码块

创建一个对象时，在一个 类调用顺序：（重点，难点）

  1. 调用静态代码块和静态属性初始化（注意：静态代码块和静态属性初始化调用的优先级一样，如果有多个静态代码块和多个静态属性初始化，则按照定义顺序调用）

  2. 调用普通代码块和普通属性初始化（注意：普通代码块和普通属性初始化调用的优先级一样，如果有多个普通代码块和多个普通属性初始化，则按照定义顺序调用）

  3. 构造器调用，最后调用构造器

  1. 构造器的最前面其实隐含了super()和调用普通代码块。

单例设计模式

单例(单个的实例)

  1. 就是采取一定的方法保证在整个的软件系统中，对某个类只能存在一个对象实例，并且只提供一个取得其对象实例的方法
  2. 单例设计模式分为饿汉式和懒汉式两种

饿汉式

  1. 构造器私有化（防止直接 new）
  2. 类的内部创建对象，不管用不用都创建
  3. 向外暴露一个公共的static方法返回一个实例对象

懒汉式

  1. 构造器私有化
  2. 定义一个static静态属性对象
  3. 提供一个public的static方法，可以返回一个Cat对象

区别

（1）创建对象时机不同，饿汉式类加载时创建，懒汉式使用时才创建
（2）饿汉式不存在线程安全问题，懒汉式存在线程安全问题。
（3）饿汉式可能浪费资源，创建了实例未使用，懒汉式是使用时才创建，不存在这个问题

抽象类

当父类的一些方法不能确定时，可以用abstract关键字来修饰该方法，这个方法就是抽象方法，用abstract来修饰该类就是抽象类

细节

  1. abstract 修饰方法，抽象方法不能有方法体

  2. 抽象类不能实例化

  3. 抽象类可以没有抽象方法

  4. 有抽象方法，必须声明为抽象类

  5. abstract只能修饰类 或者 方法

  6. 抽象方法不能有主体，不能实现

  7. 继承抽象类之后必须实现抽象类的所有抽象方法， 或者也声明为抽象类

  8. 抽象方法不能用private、final、static，因为这些和重写违背

细节

  1. 接口不能被实例化

  2. 接口中所有方法是public，接口中的抽象方法可以不用abstract修饰

  3. 一个普通类实现接口，就必须将该接口的所有方法都实现，可以用alt+enter快速实现

  4. 抽象类实现接口，可以不用实现接口方法

  5. 一个类同时可以实现多个接口

  6. 接口中的属性，只能是final的，而且是public static final 修饰符

  7. 接口中属性的访问形式：接口名.属性名

  8. 接口不能继承其他的类，但是可以继承多个别的接口

  9. 接口修饰符只能是public和默认，和类一样
